# 最終版単体計測

## 目的

旧Phase 0バンドルを削除したクリーンなホストで、最終60秒素材を組み込んだ正式版単体のCPU、RSS、終了時停止を確認する。10分連続動作は初期正式版で合格済みのため、この作業単位では素材差と旧版混在の影響を切り分ける2分計測とした。

## 条件

|項目|値|
|:---|:---|
|ブランチ|`feat/macos-video-screensaver`|
|計測日時|2026-07-16 12:57:35〜12:59:35 JST|
|macOS|26.2（Build 25C56）|
|Mac|MacBookAir10,1 / arm64|
|対象|最終 `MizuNiNaru.saver` フルスクリーンプレビュー|
|旧Phase 0|アンインストール済み|
|動画|昼 / H.264 High / 1280×720 / 約60秒 / 30fps|
|計測時間|120秒|
|間隔|5秒|
|サンプル数|24|

## 再生中

|指標|最小|平均|最大|Phase 0平均との差|
|:---|---:|---:|---:|---:|
|CPU|3.5%|6.16%|10.9%|-2.60pt（-29.7%）|
|RSS|66.83MiB|67.91MiB|68.92MiB|+10.46MiB|

RSSは開始から終了まで0.95MiB減少した。60秒ループを2回通過し、CPU上昇、メモリ増加、停止、クラッシュはなかった。

初期正式版のCPU平均7.13%と比べても0.97ポイント低い。RSS平均は初期正式版より3.58MiB高いが、旧版混在を除いた正式版単体の安定値として約68MiBを今後の基準とする。

## 終了後

|時点|CPU|RSS|
|:---|---:|---:|
|終了直後|0.0%|55.02MiB|
|終了10秒後|0.0%|54.39MiB|

macOSの待機ホストは残ったが、映像デコードと時計更新は停止し、メモリも解放された。

## 判断

合格。最終版は次を満たす。

- 初期正式版で10分連続動作に成功
- 最終60秒素材で反復感が軽減
- 正式版単体のCPU平均はPhase 0より29.7%低い
- 2分間のRSS増加なし
- システム設定終了直後と10秒後のCPUが0.0%
- 映像、日時、時計、ループの目視確認に問題なし
