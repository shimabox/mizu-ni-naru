# 4時間帯の動画生成

> これは正式実装時の初期28秒素材の記録。実機確認後の最終60秒素材は[動画の長尺化と画質比較](20260716-video-loop-quality.md)に記録している。

## 仮説

同じseed、画質、カメラ、収録開始条件で朝、昼、夕方、夜を収録し、ループ境界をクロスフェードすれば、世界の構成を変えずに時間帯を動画として切り替えられる。

## 条件

|項目|値|
|:---|:---|
|Git SHA|`b88e0b8`から開始した未コミット作業|
|代表時刻|08:00 / 12:00 / 18:15 / 21:00|
|seed|7|
|画質|`q=0`|
|DPR|1|
|解像度|1280×720|
|収録|30秒 / 30fps / VP9 WebM|
|ループ加工|先頭と末尾を2秒クロスフェード|
|出力|約28秒 / H.264 High / yuv420p / 30fps|

## 結果

|動画|時間|容量|境界PSNR|
|:---|---:|---:|---:|
|朝|28.033秒|13,934,151 bytes|34.63dB|
|昼|28.033秒|14,422,365 bytes|33.69dB|
|夕方|28.000秒|12,582,350 bytes|32.15dB|
|夜|28.033秒|6,628,118 bytes|33.43dB|
|合計|-|47,566,984 bytes|-|

境界PSNRは、H.264出力の先頭フレームと末尾フレームを比較した値。4本とも生成、デコード、目視確認に成功した。

## 判断

採用する。

- 朝、昼、夕方、夜の色と星の有無が現行WebGL画面と一致する
- 計測表示は含まれない
- 4本が同一のシミュレーション条件なので、ネイティブ切替時に再生位置を同期できる
- ループ境界は完全一致ではないが、クロスフェードなしのハードカットより差が小さい

合計約47MiBはPhase 0の夜1本約5.8MiBより増える。これは4時間帯と長いループを追加した機能コストであり、CPUと配布ZIPは正式バンドルで別途計測する。
